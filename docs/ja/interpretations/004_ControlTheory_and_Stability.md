# 004. 制御工学と安定性 (Control Theory and Stability)

このフェーズは、「組織をバラバラに引き裂くことなく、どのようにして健全な状態へと操縦（ステア）して戻すか？」という問いに答えます。TLU は高度な制御システム工学を使用して、最適な前進経路を計算し、差し迫った構造的崩壊を警告します。

---

### 1. 制御エラー収束 (`004_1_2__control_error_convergence.png`)

![004_1_2__control_error_convergence](../../readme_plots/004_1_2__control_error_convergence.png)

* **📊 視覚的構造**: 複数回の計算イテレーション（反復）にわたる、数学的な「エラー（現在の状態とターゲットとなる健全な状態との間の距離）」をマッピングした折れ線グラフ。
* **📐 物理理論**: 最適レギュレータ (LQR: Linear-Quadratic Regulator)。会社を操縦しようとする「オートパイロット（自動操縦）」をシミュレートします。このグラフは、オートパイロットが目的地に無事到達できるかどうかを示します。
* **🚨 異常検出 (Anomaly Detection)**:
  * 線が激しくバウンドする、上方にスパイクする、または下部に収束することなく、ゼロよりはるかに高い位置で横ばい（フラットライン）になる。
* **💼 ビジネスへの翻訳**: **制御不能な組織**。組織のプロセスが極めてカオスであるか、あるいは深く壊れているため、数学的最適化でさえも健全な状態に戻るための安全な経路を見つけることができません。経営陣の介入は失敗に終わるか、意図せぬ副作用を引き起こす可能性が高いです。

### 2. LQR 制御パフォーマンス空間 (`004_1_3__control_lqr_performance_space.png`)

![004_1_3__control_lqr_performance_space](../../readme_plots/004_1_3__control_lqr_performance_space.png)

* **📊 視覚的構造**: 様々な介入戦略の「コスト（Cost）」を示す3Dランドスケープまたは等高線マップ。
* **📐 物理理論**: LQR コスト関数 ($J$)。「どれだけ早く問題を修正するか」vs「その修正が組織にどれだけのひずみ（負担）をもたらすか」というトレードオフをマッピングします。
* **🚨 異常検出 (Anomaly Detection)**:
  * 滑らかで予測可能な「お椀型」ではなく、ギザギザで急峻な「渓谷（Canyon）」のような風景。
* **💼 ビジネスへの翻訳**: **高リスクの介入**。急峻なランドスケープは、経営陣の介入におけるほんの小さなミス（例: 給与を10%ではなく12%削減する）が、組織に壊滅的なひずみをもたらし、会社をカオスに突き落とすことを意味します。

### 3. システムの安定性 / スペクトル半径 (`004_1_2__system_stability.png`)

![004_1_2__system_stability](../../readme_plots/004_1_2__system_stability.png)

* **📊 視覚的構造**: 単一の指標（スペクトル半径）を追跡する時系列の折れ線グラフ。通常、決定的な赤い閾値ラインが **1.0** に引かれています。
* **📐 物理理論**: 遷移行列の固有値解析。最大固有値の絶対値が 1.0 を超えた場合、数学的システムは正式に「不安定」であり、無限大に発散します。
* **🚨 異常検出 (Anomaly Detection)**:
  * 青い線が 1.0 の閾値を突き抜け、そのまま留まる。
* **💼 ビジネスへの翻訳**: **死の螺旋（Death Spiral）**。組織は爆発的で自己強化的なフィードバックループに突入しました。これは、深刻な **循環取引（Wash Trading）**、ポンジ・スキーム、または制御不能な負債の螺旋の際に発生します。システムは自らを食い物に（カニバライズ）しており、外部からの介入が直ちに適用されない限り、数学的に自己崩壊します。

### 4. トレードオフ・ヒートマップ / 感度行列 (`004_2_1__sensitivity_matrix.png`, `004_2_2_1__tradeoff_heatmap.png`)

![004_2_1__sensitivity_matrix](../../readme_plots/004_2_1__sensitivity_matrix.png)

* **📊 視覚的構造**: ある変数が別の変数の変化に対してどれだけ敏感か（敏感度）を示すヒートマップ。
* **📐 物理理論**: 偏微分。「もし $X$ を1単位変更した場合、$Y$ はどれだけ変化するか？」
* **🚨 異常検出 (Anomaly Detection)**:
  * これまで存在しなかった場所に、極度の過敏性を示す突然の明るい色が現れる。
* **💼 ビジネスへの翻訳**: **脆弱性（Fragility）**。組織が特定の要因に対して過敏（ハイパーセンシティブ）になっています。例えば、もしマトリックスが「現金（Cash）」が「旅費交通費（Travel Expenses）」の小さな変動に対して極端に過敏になっていることを示している場合、それは会社の流動性バッファーが消失しており、わずかな過剰支出であっても直ちにキャッシュ危機を引き起こすことを意味します。
