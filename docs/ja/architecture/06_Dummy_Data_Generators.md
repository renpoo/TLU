# 06. ダミー・データ・ジェネレーター (Dummy Data Generators)

> **"診断エンジンの真価は、それが検出できる異常の質によってのみ決まる。"**

Tensor-Link Utility (TLU) の分析力を検証するために、システムには2つの高度なダミー・データ・ジェネレーターが組み込まれています。これらは単なる乱数発生器ではありません。複雑な現実世界のネットワークのダイナミクスをシミュレートし、TLUの検出機能の限界をテストするために「数学的に正確な異常（ノイズや毒）」を注入するように設計されています。

---

## 1. ダミー仕訳ジェネレーター（財務・組織のフロー）
**スクリプト:** `src/filters/_0_0_generate_dummy_journal.py`  
**実行コマンド:** `bash bin/batch_generate_dummy_journal_data.sh`

このジェネレーターは、中規模企業の複数年にわたる日々の会計仕訳をシミュレートします。部門（営業、R&D、運用など）や口座（現金、売掛金、買掛金など）をネットワーク内のノードとしてモデル化します。

### 自然なベースライン・ダイナミクス
* **スケールフリー・ネットワーク・トポロジー:** 取引はべき乗則分布に従います。少数のノード（`Cash` や `Accounts_Payable` など）が巨大なハブとして機能し、他のノードは周辺に位置します。
* **因果イベントキュー（粘性）:** 今日の売上（`Sales -> AR`）は、将来（30〜60日後）の回収イベント（`AR -> Cash`）をスケジュールします。この時間的遅延が、位相空間の力学における自然な「粘性（Viscosity）」を生み出します。
* **ピンクノイズ（1/f ノイズ）:** 人工的な「平坦さ」を避けるため、取引ボリュームはフラクタル・ノイズによって変調され、自然な景気循環やマクロ経済のトレンドを模倣します。

### 異常注入メカニズム
ジェネレーターは、TLUのフォレンジック・フィルターやトポロジー・フィルターをトリガーするために、特定の構造的およびボリュームの異常を意図的に注入します：

1. **Z-Spike（外れ値の支出）:** 突然発生する巨大な1回限りの取引。標準的なZ-Score分析（ミクロ・フォレンジック）によって簡単に捕捉されます。
2. **Leak（横領・保存則の喪失）:** 相対する（バランスする）入力なしに、資金がシステムから外部へ移動します。マクロ・フォレンジックの「保存則のリーク」検出器によって捕捉されます。
3. **Drift（静かなサプライヤーの切り替え）:** ある部門の総支出は完全に正常なままですが、資金の*送信先*が突然新しいベンダーにシフトします。Z-Scoreでは不可視ですが、**局所的KLダイバージェンス・ドリフト（Local KL Drift）** に巨大なスパイクを生み出します。
4. **Unbalanced Mistake（仕訳エラー / 外部への漏出）:** 借方 $\neq$ 貸方となる取引が生成されます（例：データの喪失や、元帳の境界外への資金の消失）。これはキルヒホッフの電流則に違反しており、マクロ・フォレンジックの **保存則残差（Conservation Residual）** センサーによって即座に検出されます。
5. **Wash Trading（循環取引）:** 意図的に閉ループ（例：`Cash -> AR -> Sales -> Cash`）を形成するように設計された一連の取引。これは会計元帳の自然な有向非巡回グラフ（DAG）構造を破壊し、**システム安定性（スペクトル半径）** フィルターによって即座にフラグが立てられます。

## 2. ダミー・トラフィック・ジェネレーター（物理的・物流のフロー）
**スクリプト:** `src/filters/_0_0_generate_dummy_traffic.py`  
**実行コマンド:** `bash bin/batch_generate_dummy_traffic_data.sh`

仕訳ジェネレーターが会計に焦点を当てているのに対し、このジェネレーターは京都の街路網（例：四条烏丸）のような格子状の交差点モデルに基づいて、物理的なオブジェクトのフローをシミュレートします。

### 特徴
* **空間グリッド・ネットワーク:** ノードは物理的な交差点を表し、エッジは道路の区間を表します。フローは物理的に隣接するノードに制限されます。
* **集中型グラビティ（重力）:** 交通量は自然にグリッドの中心（京都市内中心部をシミュレート）に集まり、情報曲率多様体（Information Curvature manifold）に急峻な「谷」を生成します。
* **渋滞シミュレーション:** 構造剛性（ある交差点でのショックが隣接する交差点にどのように波及するか）や多様体の次元性（交通渋滞がネットワークの有効ランク（Effective Rank）をどのように崩壊させるか）のテストを可能にします。

このジェネレーターは、TLUの数学的抽象化（質量、粘性、エントロピー）がドメインに依存せず、財務元帳と全く同様に物理的なロジスティクスにも適用できることを証明します。
