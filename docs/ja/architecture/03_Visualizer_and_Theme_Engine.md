# 03. ビジュアライザーとテーマエンジン (Visualizer and Theme Engine)

> **"数学は真実を発見し、可視化はそれを否定できないものにする。しかし、歪んだレンズは最も純粋な方程式でさえも歪曲してしまう。"**

TLU パイプラインの第3フェーズ（Phase 3: プレゼンテーション）は、認知負荷のオフロード（軽減）を担当します。純粋数学フィルター（カテゴリ 000-004）によって生成された高密度な CSV ストリームを、高精度の視覚的アーティファクト（成果物）へと翻訳します。

しかし、TLUにおける可視化は単なる美観の問題ではありません。視覚的な表現が背後にある数学的真実を決して裏切らないことを保証するための、厳格なアーキテクチャの法則に支配されています。

---

## 1. 最優先指令：ビューでの計算禁止

Visualizer 層は、いかなる分析的な計算も実行することを厳格に禁じられています。

* Z-Scoreを計算することはなく、平均値を算出することも、タイムラグを決定することもありません。
* その唯一の責任は **レンダリング（Rendering）** です。事前に計算された数値を視覚的な次元（色、サイズ、形状、Z軸）に射影することです。入力ストリームからある指標が欠落している場合、ビジュアライザーはクラッシュします。自らその指標を計算しようとは決してしません。この厳格な分離により、視覚的な出力が数学的コアの完全な鏡（1:1）であることが保証されます。

## 2. フェイルファスト・テーマエンジン

多くの場合、ダッシュボードは経営幹部の意思決定における主要なツールとなります。もしカラーコードが欠落していたり、暗黙のうちに適用されたデフォルトカラーによってチャートがデータを誤って表現したりすれば、その結果としてのビジネス上の決定は破滅的なものになる可能性があります（コンテキスト・ブリード）。

これを防ぐため、TLU は **フェイルファスト・テーマエンジン** を採用しています。

* **辞書の厳格な強制:** テーマ（例：`dark`, `light`, `colorblind-safe`）が読み込まれる際、エンジンは必要なすべてのカラーキーとカラーマップが存在するかを検証します。
* **フォールバック・ゼロ:** 特定のノードタイプに割り当てられた意味論的な色（Semantic Color）が欠落している場合、ビジュアライザーはデフォルトの「グレー」にフォールバックすることはありません。`KeyError` をスローし、パイプライン全体を即座に停止させます。嘘をつくダッシュボードを出力するくらいなら、壊れたパイプラインの方がマシだからです。

## 3. 意味論的カラーマッピング：ネットワーク・トポロジーとストレス

TLUの視覚的哲学の最たる例が、**ネットワーク・トポロジーとストレス伝播**ビジュアライザー（`002_1_2: Topological Edge Stress` に対応）です。時間が経過するにつれて、組織のストレスは血管を流れる血液のようにネットワークを流れていきます。TLU は、デュアル（二重）・カラーマップシステムを使用してこれを捉えます：

### ノードのレンダリング（純フラックス）

* **指標:** 絶対的な流入量 vs. 流出量 ($q_i$)。
* **カラーマップ:** 分岐カラーマップ（Diverging Colormap / 例：`Coolwarm` - 濃い赤から濃い青へ）。
* **意味（セマンティクス）:** ノードの現在の「質量」と極性を特定します。エネルギーを急速に蓄積しているのか（青）、それとも出血しているのか（赤）？

### エッジのレンダリング（位相幾何学的ストレス）

* **指標:** 接続を通過するフラックスのZ-Score。歴史的基準からの逸脱を表します。
* **カラーマップ:** 順次的で知覚的に均一なカラーマップ（Sequential Colormap / 例：`Viridis` または `Plasma` - 濃い紫から明るい黄色へ）。
* **意味（セマンティクス）:** 濃い紫のエッジは正常で健康的な流れを示します。Z-Scoreが正常な閾値を超えて上昇すると、エッジは太くなり、明るい緑から黄色へと変化します。これにより人間は、前例のない負荷がかかっている経路、すなわち**隠れたボトルネック**や**破裂リスク**を瞬時に発見することができます。

### 時系列の伝播（脈動）

各タイムステップ（`t_idx = 0, 1, 2...`）ごとに個別のフレームを生成することで、アナリストは組織の「脈動（Pulse）」を観察することができます。あるノードへのショック（例：R&D支出の大規模な急増）は、その後のタイムステップにおいて下流のエッジへと伝播していくストレスの黄色い波として視覚的に追跡することができ、運動学（Kinematics）層によって計算された因果の波及効果を実証的に証明します。

## 4. ユニバーサルな 3D 曲面翻訳

位相空間、熱力学的複雑性、および情報曲率について、TLUは 3D 多様体（Manifold）レンダリングに大きく依存しています。

* **X/Y 軸:** 空間トポロジーまたは時間を表します。
* **Z 軸 (高さ):** 幾何学的歪み（曲率）、ひずみエネルギー、または速度を表します。
* **表面色 (Surface Color):** 独立した指標（密度や温度など）を重ね合わせ、単一の 3D 表面上に第4の次元の洞察を提供します。

これらの厳格で意味論的にマッピングされた視覚エンジンを通じて、TLU はドメインの複雑性が絶対的な数学的——および視覚的——透明性へと射影されることを保証します。

## 5. インタラクティブ探索モード（GUI 検査）

高次元のデータ、特に 3D 位相空間や密集したネットワーク・トポロジーを可視化する場合、隠された洞察を発見するためには手動での検査が必要になることがよくあります。TLU のデフォルトはヘッドレスの高速バッチ・レンダリングモード（自動的にPNGを保存して終了する）ですが、ネイティブで **インタラクティブモード** をサポートしています。

* **呼び出し方法:** 任意の可視化スクリプトまたはメインのバッチ・オーケストレーターに `--interactive` フラグを追加する（`bash bin/batch_visualize_graphs.sh dark --interactive`）と、パイプラインは各フレームを生成した後に動的に一時停止します。
* **機能:** これにより、ネイティブの Matplotlib GUI ウィンドウが起動し、アナリストはマウスを使用して以下の操作を行えます：
  * **3D 表面の回転:** 位相ポートレイトや熱力学の多様体を自由に回転させ、最適な視野角を見つけます。
  * **パンとズーム:** 密集したネットワーク・トポロジーのクラスターへと潜り込みます。
* **パイプラインの統合:** システムは、画像が保存されたり次の数学的フレームへ進んだりする前に、ユーザーが GUI ウィンドウを閉じるのをインテリジェントに待ちます。人間の直感と自動バッチ処理をシームレスに融合させます。
