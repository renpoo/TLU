# 02. データのトポロジーと認識論的射影 (Data Topology and Epistemological Projection)

> **"システムの真実を計算するためには、まず名前という幻想を剥ぎ取らねばならない。"**

TLU がデータに物理学や幾何学の法則を適用する前に、データからドメイン（業務領域）特有のコンテキストをパージ（消去）する必要があります。アルゴリズムは「買掛金（Accounts Payable）」「マーケティング部門」「米国東部サーバーリージョン」といった言葉を理解しません。アルゴリズムが理解するのは、トポロジー（位相）、質量、および方向性のあるフラックス（流量）のみです。

このドキュメントでは、生のドメインデータを純粋な数学空間へと変換するための厳格なプロトコルを定義します（Ver 8.0.0）。

---

## 1. グレート・フィルター（ドメインの不可知論）

TLU パイプラインの最初のフェーズは **Phase 1: 射影（Projection）** です。その唯一の目的は「認識論的射影」です。つまり、厄介で人間が読める世界を、無菌で計算可能なテンソル空間へと翻訳することです。

* **語彙の剥奪:** データがプロジェクター (`_0_2_projector_to_coo.py`) を通過すると、すべての文字列ベースの識別子とドメインロジックが根絶されます。
* **COO テンソルストリーム:** この時点以降、パイプラインは標準的な Coordinate (COO) ストリームフォーマットでのみデータを受け入れ、出力します：
  `t_idx, src_idx, tgt_idx, value`
  *(時間インデックス, 送信元ノードインデックス, 宛先ノードインデックス, フラックス量)*
* **数学的純粋性:** コア解析フィルター（Phase 2）は意図的に「盲目（blind）」にされています。それらは、自分が財務元帳を分析しているのか、生体ニューラルネットワークを分析しているのか、あるいは物流のサプライチェーンを分析しているのかを知りません。このドメインの不可知論（Agnosticism）が、人間のバイアスなしに数学的ルールが適用されることを保証します。

## 2. ユニオン・トポロジー（次元の整合性）

時間経過にわたって行列表算（逆共分散行列の計算など）が数学的に有効であるためには、空間の次元が完全に一貫していなければなりません。

* **一貫した空間座標:** ノードに割り当てられた整数ID（例: `Node 4`）は、すべてのタイムステップ（`t_idx = 0` から `t_idx = N` まで）にわたって、正確に同じエンティティ（実体）を表していなければなりません。
* **全要素の和集合（Union）:** ある特定の部門や口座がデータセットの最後の月にしか現れない場合でも、それは最初からトポロジー空間に登録されます。TLU は「ユニオン・トポロジー（和集合位相）」を構築します。これは観測されたタイムライン全体で*かつて存在した*一意のノードの総数を $N$ とした、$N \times N$ サイズの空間行列です。欠落しているフラックスは、単にゼロエネルギーの転送として扱われます。

## 3. エフェメラルな辞書（人間世界へのリンク）

コア・パイプラインがすべての人間用のラベルを破壊してしまうため、TLU は最終的な数学的結論をビジネスの洞察へと翻訳し直す方法を維持しなければなりません。

* **使い捨てのレジストリ:** 射影（Projection）フェーズ中に、TLU は2つのエフェメラル（一時的）なアーティファクト、`_node_map.csv` と `_time_map.csv` を生成します。
* **動的再構築:** これらの辞書は永久に保存されるデータベースではありません。これらは使い捨てであり、プロジェクターが実行されるたびにゼロから再構築されます。これにより、ソースデータのカテゴリが変更されても、スキーマのマイグレーションなしでシステムが即座に適応することが保証されます。
* **Phase 3 での再アタッチ:** これらの辞書は、パイプラインの最後（Phase 3 の Visualizer 層）でのみ再び参照され、3Dグラフやダッシュボードに人間が読める名前でラベル付けを行います。

## 4. フェイルファストな列マッピング

データの取り込み（Ingestion）は、あらゆる分析パイプラインにおいて最も危険な障害発生ポイントです。TLU はデータ取り込みを極めて厳格に（妥協なく）処理します。

* **厳格な要件:** プロジェクターは、ソースCSVのどの列が 時間（Time）、送信元（Source）、宛先（Target）、値（Value）を表すかを定義する明示的なCLI引数を要求します。
* **推測なし、フォールバックなし:** 指定された列が欠落していたり激しく破損していたりする場合、TLU はユーザーの意図を推測したりデフォルト値を適用したりしようとはしません。即座に `KeyError` をスローし、パイプラインをクラッシュさせます。この **フェイルファスト（Fail-Fast）** 設計により、汚いデータが入り口で捕捉され、それが下流の複雑な物理計算を気付かないうちに汚染する（コンテキスト・ブリード）ことを防ぎます。
